## 4. PhoneMockup Component — Real Screenshot Support

Updated `src/app/page.tsx` `PhoneMockup` to accept an optional `imageSrc` prop:

```tsx
<PhoneMockup screen="scan" imageSrc="/images/screens/scan.png" />
// When imageSrc is omitted → falls back to coded UI (existing behavior)
```

**How to swap in real screenshots**:
1. Take screenshots on iPhone Simulator (Xcode → Device → File → Take Screenshot)
2. Run through Apple Frames (free Figma plugin) to add device chrome, OR use Rottenwood / Mockuphone
3. Export as PNG, place in `public/images/screens/`:
   - `public/images/screens/scan.png`
   - `public/images/screens/compare.png`
   - `public/images/screens/dashboard.png`
   - `public/images/screens/report.png`
4. Pass `imageSrc` prop to each `<PhoneMockup>` call in `src/app/page.tsx`

The component renders `<Image fill className="object-cover object-top" />` inside the phone frame (below the status bar notch), sized correctly for the `w-56 sm:w-64 aspect-[9/19]` frame.

## 6. Architecture Notes

### Token reference (tailwind.config.ts)
| Intent | Token | Value |
|---|---|---|
| Nav / dark backgrounds | `bg-navy` | `#0D0B28` |
| Brand blue accent | `navy-500` | `#271881` |
| Primary CTA / highlights | `coral` | `#00CA5A` (green) |
| Secondary accent | `blue` | `#03C0DE` (cyan) |
| Light section alt bg | `cream` | `#F2F5FF` |
| Card shadow hover | `shadow-card-hover` | `0 8px 32px rgba(13,11,40,0.18)` |

### Button variants
- `primary` — green (`bg-coral`) — primary CTA
- `outline` — navy border — secondary on white/light backgrounds
- `outline-white` — white border — secondary on dark/navy backgrounds
- `ghost` — no border, text only

### Section component
```tsx
<Section background="white">   // bg-white
<Section background="cream">   // bg-cream (F2F5FF)
<Section background="navy">    // bg-navy text-white
```

### Framer Motion pattern
```tsx
// All animated sections use:
<FadeUp delay={0.1}>  // once: true, so no re-trigger on scroll-up
// useReducedMotion is checked in FadeUp component
```
